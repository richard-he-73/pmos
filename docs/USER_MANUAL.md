## 系统概述

PMOS (Project Management Operating System) 是一个面向 PMO 的企业管理系统，提供项目全生命周期管理能力。

**核心特性**:
- 📊 项目全生命周期管理
- 📋 任务分配与跟踪
- 👥 资源负载监控
- ⚠️ 三级预警体系
- 📈 数据可视化与报表导出

---

## 快速开始

### 1. 登录系统

1. 打开浏览器访问 `http://localhost:16001`
2. 输入用户名和密码登录
3. 首次使用请注册新账号

### 2. 创建第一个项目

1. 点击左侧导航栏「项目管理」
2. 点击「新建项目」按钮
3. 填写项目基本信息（名称、编号、时间、预算）
4. 点击「保存」

### 3. 添加任务

1. 进入项目详情页
2. 点击「任务」Tab
3. 点击「新建任务」
4. 填写任务信息并分配负责人

## 核心功能

### 项目管理

#### 项目列表
- 支持表格视图和卡片视图
- 可按状态筛选（规划中/进行中/已完成/已暂停/已归档）
- 支持项目编号、名称搜索

#### 项目创建/编辑
- **必填字段**: 项目编号、名称、开始日期
- **可选字段**: 描述、结束日期、预算、标签
- 项目编号不能重复

#### 项目状态流转
- 状态流转遵循: 规划中 → 进行中 → 已完成 → 已归档
- 可从任意状态暂停（→ 已暂停）
- 暂停后可恢复（→ 进行中）

#### 项目克隆
- 点击项目详情中的「克隆」按钮
- 克隆后的项目状态重置为「规划中」
- 进度和预算重置

### 任务管理

#### 任务视图
- **表格视图**: 适合详细任务管理
- **看板视图**: 按状态分组，拖拽更新状态

#### 任务属性
- **状态**: 待办 → 进行中 → 审核 → 已完成
- **优先级**: 低 / 中 / 高 / 紧急
- **类型**: 功能 / Bug / 任务 / 里程碑
- **估算工时/实际工时**: 用于工作量统计

#### 任务依赖
- 可为任务设置前置依赖
- 系统自动检测循环依赖并阻止
- 依赖关系在甘特图中可视化显示

### 资源管理

- 资源类型: 人力资源、设备、场地、软件
- 负载监控: 容量 vs 已分配比例
- 冲突检测: 自动检测同一人员在时间上重叠的任务

### 风险管理

- 风险严重度: 低 / 中 / 高 / 严重
- 风险概率: 0.0 - 1.0
- 影响评分: 概率 × 严重度
- 支持风险应对策略记录

### 需求管理

- 需求类型: 功能需求 / 非功能需求 / 业务需求
- 需求状态: 草稿 → 评审 → 批准 → 实现 → 验证 → 完成
- 需求与任务关联追踪

### 测试管理

#### 测试用例
- 完整的测试用例管理（CRUD）
- 模块分类、优先级、状态管理
- 支持标签和前置条件

#### 缺陷追踪
- 严重度: 低 / 中 / 高 / 严重
- 状态: 新建 → 已打开 → 修复中 → 已解决 → 已关闭
- 环境记录和复现步骤

#### 测试报告
- 自动统计通过/失败/阻塞数量
- 通过率计算
- 缺陷关联统计

## 高级功能

### 项目状态流转

**状态流转图**:
```
[规划中] ──启动──→ [进行中] ──完成──→ [已完成] ──归档──→ [已归档]
   │                  │                    │
   └───暂停──→ [已暂停] ──恢复──→ [进行中]    └──归档──→ [已归档]
```

**操作方式**:
1. 进入项目详情页
2. 点击「操作」菜单选择「状态流转」
3. 选择目标状态并填写原因
4. 确认后完成状态变更

**状态变更时自动操作**:
- 规划中 → 进行中: 自动记录 started_at
- 任何状态 → 已完成: 进度设为 100%
- 任何状态 → 已归档: 自动记录 archived_at

### 任务依赖关系

**API 端点**:
- `GET /tasks/{id}/dependencies` - 查看任务依赖
- `POST /tasks/{id}/dependencies` - 添加依赖
- `DELETE /tasks/{id}/dependencies/{dep_id}` - 移除依赖

**注意事项**:
- 不能依赖自身
- 系统自动检测并阻止循环依赖
- 依赖任务不存在时无法添加

### 预警中心

**预警类型**:

| 类型 | 触发条件 | 预警级别 |
|------|----------|---------|
| 超期任务 | 任务截止日期已过但未完成 | 🔴 严重 |
| 临近截止 | 3 天内到期的未完成任务 | 🟡 警告 |
| 预算超支 | 已用预算 > 80% 总预算 | 🟡 警告 |
| 项目延期 | 进行中项目但结束日期已过 | 🔴 严重 |
| 高风险 | 风险概率 ≥ 0.7 且未关闭 | 🔴 严重 |
| 严重缺陷 | 严重度为 critical 且未解决 | 🔴 严重 |

**API 端点**: `GET /resources/alerts`

## 常见问题

### Q: 为什么项目无法删除？
A: 已归档的项目才能删除，进行中的项目请先归档。

### Q: 如何设置任务依赖？
A: 在任务详情页的「依赖关系」Tab 中添加前置任务。

### Q: 预算超支预警如何设置阈值？
A: 默认 80%，可在系统设置中调整。

### Q: 如何导出报表？
A: 进入「导出报表」页面，选择项目、报表类型和时间范围，然后导出。
